01

仕組み — 2 つの部品

取
PROVISIONING CLI

takaichi-blog-fetch

初回に一度だけ実行。Wayback Machine から全記事をダウンロードし、あなたの PC に Markdown 形式で保管します（生 HTML は保存しません）。各記事には出典 URL を付与。

供
MCP SERVER · stdio

takaichi-blog-mcp

Claude Desktop などがローカルで起動。保存済みデータのみを参照するためネットワークアクセスは不要で、高速かつ安定して動作します。

一括ダウンロードは約 1,024 リクエスト（約 20 分）かかるため、サーバ起動処理には含めていません。 MCP クライアントの初期化タイムアウトを避けるための設計です。

02

インストール

zsh — takaichi-blog-mcp ⧉ copy
# 1. PyPI からインストール（クローン不要） $ pip install takaichi-blog-mcp # 2. ブログ記事を一括ダウンロード（初回のみ・約20分） $ takaichi-blog-fetch Fetching CDX index from Wayback Machine ... Found 1024 archived articles. Fetching 1024 articles (~20 min) ... [1024/1024] fetched (ok=1024) Done. 1024/1024 articles saved. # 3. 動作確認（通常は MCP クライアントが自動起動） $ takaichi-blog-mcp --help

pipx / uv を使う場合（推奨）: コマンドを隔離環境にインストールできます — pipx install takaichi-blog-mcp / uv tool install takaichi-blog-mcp。 Python 3.10 以上が必要です。

03

提供する MCP ツール

takaichi_list_articles
記事メタデータの一覧。カテゴリ・公開日で絞り込み、新しい順、ページング対応。
takaichi_list_categories
カテゴリ一覧（件数・期間付き）。各カテゴリは当選期に対応します。
takaichi_get_article
ID を指定して記事本文（Markdown）とメタデータを取得。出典 URL 付き。
takaichi_search_articles
タイトル＋本文の全文検索（部分一致・日本語対応）。一致箇所の抜粋を返します。
04

Claude Desktop に登録

claude_desktop_config.json
{ "mcpServers": { "takaichi-blog": { "command": "takaichi-blog-mcp" } } }

設定ファイルに上記を追記して Claude Desktop を再起動するだけ。事前に takaichi-blog-fetch でデータをダウンロードしておく必要があります。

  • 出典の明示 — 各記事に元サイト URL と Wayback URL を保持
  • ローカル完結 — 起動後はネットワーク不要
  • 日本語全文検索 — 依存ライブラリなしの部分一致
